Executive Summary

A spring seasonal email built around sandals and warm-weather footwear. The hero lands cleanly with a strong discount offer and "Spring Staples" positioning, but the product grid below it is thin — three items visible in a sparse layout that feels like it was designed for more inventory. The secondary navigation and utility modules are present and functional, but they pad out the bottom half without adding real purchase intent. The email does its job, but undersells the depth of the catalog.

Business Impact Score: 6/10

What's Working

  • Hero is tight. "50% OFF" badge, "Spring Staples" headline, and a lifestyle sandal image cohesively communicate the seasonal offer. The CTA ("SHOP NOW") is prominent and positioned immediately below the offer.
  • Product selection is appropriate. The featured sandals — espadrille-style flats and strappy slides — are visually on-brand and seasonally relevant.
  • Category nav is thorough. WOMEN / MEN / KIDS / CLOTHING / NEW ARRIVALS / SALE gives the recipient a clear escape hatch to browse regardless of whether the hero resonates.
  • Utility row is complete. Download the App, Afterpay, Curbside Pickup, and Find a Store are all represented with legible icons and labels.

What's Weak

  • Product grid is too sparse. Only three products appear below the hero, arranged in a 2+1 layout with significant whitespace. This reads as underfilled — not curated.
  • "SHOP ALL BOBS" CTA is unexplained. The sub-brand "BOBS" appears in a CTA without any introduction or context in the hero or surrounding copy. A recipient unfamiliar with the brand gets no explanation.
  • "Best Selling Styles Going Fast" module is undercut by its own sparseness. A single "SHOP NOW" CTA with no product imagery directly contradicts the urgency of the headline. Urgency claims need visual proof.
  • "SHOP ADDITIONAL STYLES" section is essentially just a text nav. No imagery, no product callouts — just category links that duplicate what's already in the header nav. This wastes real estate.
  • Subject line is raw test metadata. [Test]: MKG_US_NA_W_MULTI_GENDER_EN_03282026_A_V3 is fully exposed in the from/subject — a production risk if the email went live this way.
  • Preview text is broken. The preview shows raw URLs and token strings, not a human-readable teaser. A missed opportunity and a trust signal problem.

Recommendations

1. Expand the product grid to 4–6 items. Fill the visible gap below the hero with more sandal/spring options. Even a 2-column, 3-row layout would feel intentional rather than incomplete.

2. Introduce BOBS before the CTA. A one-line callout ("Shop our BOBS collection — comfort meets cause") would convert the orphaned CTA into a meaningful sub-campaign.

3. Add product thumbnails to the urgency module. "Best Selling Styles Going Fast" needs 3–4 product images to earn its headline. Without them, the urgency is hollow.

4. Remove or consolidate the text-only category nav. It duplicates the header and adds no visual value. Replace it with a lifestyle image or an additional product module.

5. Fix the subject line and preview text before any live send. The test tag and raw URL preview are unacceptable for recipient-facing deployment.

Bottom Line

The bones are solid — good seasonal hook, correct product type, functional utility footer. But the email is underbuilt in its middle section. The product grid doesn't deliver on the promise of "Spring Staples," the urgency module has no teeth, and the text nav is filler. A stronger version of this email would trim the redundant nav, fill out the product grid, and give BOBS a proper introduction. As-is, it's a passable seasonal send that leaves conversion on the table.

## Technical Audit — Skechers BOGO Spring Styles (MKG_US_NA_W_MULTI_GENDER_EN_03282026_A_V3)

1. Technical Summary

This is a test-flagged promotional email with 16 images served over insecure HTTP and a malformed Return Path pixel URL. Authentication header status is unverified, which blocks deliverability confidence assessment.

2. Link & Tracking Issues

Return Path pixel — broken URL (missing protocol)

The pixel is rendered without a scheme, making it a relative path rather than an absolute URL. It will silently fail to load.

`

src="pixel.app.returnpath.net/pixel.gif?r=964ce1dbf0dcef79cbad82c12fe25342701e4499"

`

Fix: prepend https://.

ink1000.com tracking pixel — HTTP

`

src="http://www.ink1000.com/p/up/7076d8b15a776085/o.gif?mi_u=9015215849&mi_ecmp=MKG_US_NA_W_MULTI_GENDER_EN_03282026_A"

`

This pixel will be blocked in Gmail, Outlook.com, and other HTTPS webmail environments. Upgrade to HTTPS or confirm with the vendor that an HTTPS endpoint exists.

26 click-tracking links — UTMs unverifiable

All click URLs use encrypted query strings (click.emails.skechers.com/?qs=eyJ...). UTM parameter correctness cannot be confirmed from source inspection. See §6.

Krux DMP beacons — expected

beacon.krxd.net pixels (user-match + impression) are present and appear correctly resolved with subscriber hashed identity and campaign ID. No issues.

3. Rendering & Accessibility

16 images served over HTTP — mixed content risk

All hosted assets on image.emails.skechers.com use http:// rather than https://. Modern webmail clients (Gmail, Outlook.com) proxy images through HTTPS; other clients or strict CSP environments may block them entirely, producing broken image display.

Affected sources (representative list):

  • http://image.emails.skechers.com/lib/fe3115707564047a731c78/m/1/dde00662-169f-447d-b0e2-fc65f6c2290c.png (Skechers logo)
  • http://image.emails.skechers.com/lib/fe3115707564047a731c78/m/11/0ec56c9f-eec2-471a-898d-0ca1cea7e115.png (App Store badge)

  • Social icons: Instagram, Twitter, Facebook, Pinterest, YouTube — all HTTP
  • 3 additional unnamed images (UUIDs: 49468f73, 00100b23, fc08601a)

5 images missing alt text

Images without alt text will render as blank boxes when images are disabled:

  • o.gif (ink1000 tracking pixel — alt="" is acceptable for a 1×1 pixel, but flagged by QA)
  • 49468f73-4651-4af3-bea2-61d1ae5db486.png
  • 00100b23-0afa-4715-a8cd-23bb46f9d2b2.png
  • fc08601a-fcce-4c96-8b90-39e5a55d6c05.png

The 1×1 tracking pixel (o.gif) having alt="" is technically correct; the three content images above need descriptive alt text.

maximum-scale=1 in viewport meta

`html

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1" />

`

Disabling user zoom fails WCAG 1.4.4 (Resize Text). Remove maximum-scale=1.

-webkit-text-size-adjust: none

Applied globally via * { -webkit-text-size-adjust: none; }. This suppresses automatic text scaling on iOS, degrading readability for low-vision users. Use 100% instead of none.

5. Compliance

Subject line contains [Test]: prefix — critical for production

`

Subject: [Test]: MKG_US_NA_W_MULTI_GENDER_EN_03282026_A_V3Fresh Spring Styles...

`

This prefix must be stripped before any live audience deployment. The internal campaign code (MKG_US_NA_W_MULTI_GENDER_EN_03282026_A_V3) is also exposed in the subject — this is not subscriber-facing subject copy.

Authentication headers — unverified

The QA relay did not capture Authentication-Results headers, so SPF/DKIM/DMARC pass/fail status is unknown for this test send. Confirm SPF alignment (emails.skechers.com sender domain) and DKIM signature before production deployment.

CAN-SPAM / unsubscribe

Physical mailing address and unsubscribe mechanism are not visible in the truncated source but are expected in the footer. This must be verified in the full HTML before deployment — if absent, the email is non-compliant with CAN-SPAM §5.

6. Email-to-Site Continuity

UTM parameters — unverifiable from source

All 26 click URLs are encrypted (click.emails.skechers.com/?qs=eyJ...), making it impossible to confirm whether UTM parameters (utm_source, utm_medium, utm_campaign, utm_content) are correctly set on destination landing pages. The campaign code MKG_US_NA_W_MULTI_GENDER_EN_03282026_A is present in the Krux impression beacon and ink1000.com pixel, suggesting it is being used as the campaign identifier — verify it maps to the correct analytics campaign tag.

A/B variant tracking

The _A suffix in the campaign ID indicates this is variant A. Confirm variant B (_V3 or _B) uses a distinct utm_content value to avoid blended attribution in reporting.
